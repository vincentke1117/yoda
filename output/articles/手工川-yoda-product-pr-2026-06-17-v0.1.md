# Yoda · 致每一个被 Agent 折磨过的开发者（宣言版）

我知道，你们很累。又一个模型登顶了，又一个 agent 刷屏了，又一串新名词冒出来了，我知道。我啊，我累的不是这个。这些日子我想得很多：我们手里明明攥着最聪明的模型，可怎么一次又一次地，干起活来还是这么累、这么憋屈呢？这个问题不解决，你就永远在给工具打工，所谓"超级个体"，就永远是一个泡影。

那么，我们真正想要的 AI 产品，究竟该是什么样子？

**它应该是统一的。** 可 AI 三年来，我们看到的是什么？是 Claude Code 开一个终端、Codex 开一个终端、Gemini 又开一个终端，同一件事在七八个窗口里各说各话。Yoda 把 31 种客户端收进同一个工作区——按任务挑选、随时恢复会话、跑对比、互相 review；Linear、Jira、GitHub 的工单直接落进会话，CI/CD 状态就摆在 diff 旁边。一个工作区，而不是一地鸡毛。

**它应该是自由的。** 自由，是开发者天赋的权利！可 AI 三年来，我们看到的是什么？是范式一套接一套地冒出来——TDD、Spec、BMAD、superpower……每一套都自称唯一正解，每个工具都把自己信的那套焊死，逼你照它的规矩来。Yoda 不替你选：你信哪套，就用哪套；想换，随时换。主分支直跑求快，独立 worktree 求稳，brainstorm、compare、review、team 多路并进——是你挑范式，不是范式挑你。

**它应该是透明的。** 人人都说"提示词为王"，可提示词只是水面上的一层；agent 真正的行为，是它加载的 Skills、Hooks、Memory 和上下文决定的。可 AI 三年来，我们看到的是什么？是出了问题，你只能对着黑盒猜：是模型变笨了，还是我哪里没写对？Yoda 把这条链路整个摊开——逐项查看、按需覆写、持久化。多数"agent 怎么突然不行了"，根子不在模型，就在这层没人给你看的 harness 里。

有人要说我偏执：做个工具，张口闭口都是 harness。是的，我这些年就认这一件事——harness。我没有别的指望，就一个指望：让它不再是一个被忽略的术语、一句"模型决定一切"的空话，而成为你每天实实在在的工作方式，成为你和你那队 agent 之间牢不可破的默契。

这，就是 Yoda：**Your Orchestra of Delegated Agents**——你那支被妥善指挥的 agent 乐队。我深信，总有一天，"指挥一队 agent 干活"会像今天打开编辑器一样，天经地义。

不绑模型，不绑厂商，本地优先。这是潮流，浩浩荡荡——顺之者得心应手，逆之者疲于奔命。这一天，不必再等，就从这里开始：
