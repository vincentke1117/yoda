# Yoda · 致每一个被 Agent 折磨过的开发者（宣言版）

> 体例说明：本文为 Yoda 产品发布宣言，采用演说体，语言风格取法《走向共和》
> 大结局孙中山"共和演讲"——以"我知道"开篇，以"可 AI 三年来，我们看到的
> 是什么？"为排比鼓点层层推进。

---

我知道，你们很累。又一个模型登顶了，又一个 agent 刷屏了，又一串新名词冒出来了，我知道。我啊，我累的不是这个。我们手里明明攥着最聪明的模型，可干起活来，怎么还是这么累、这么憋屈？这个问题不解决，你就永远在给工具打工，所谓"超级个体"，永远是个泡影。

那么，我们真正想要的 AI 产品，究竟该是什么样子？

**它应该是统一的。** 可 AI 三年来，我们看到的是什么？Claude Code 一个终端、Codex 一个终端、Gemini 又一个终端，同一件事在七八个窗口里各说各话。Yoda 把 31 种客户端收进同一个工作区，挑选、恢复、对比、互审，工单和 CI/CD 状态就摆在 diff 旁边。一个工作区，而不是一地鸡毛。

**它应该是自由的。** 自由，是开发者天赋的权利！可 AI 三年来，我们看到的是什么？是范式一套接一套地冒出来——TDD、Spec、BMAD、superpower……每一套都自称唯一正解，每个工具都把自己那套焊死，逼你照它的规矩来。Yoda 不替你选：你信哪套，就用哪套，想换随时换。是你挑范式，不是范式挑你。

**它应该是透明的。** 人人都说"提示词为王"，可提示词只是水面上的一层；agent 真正的行为，是它加载的 Skills、Hooks、Memory 和上下文决定的。可 AI 三年来，我们看到的是什么？出了问题，你只能对着黑盒猜：是模型变笨了，还是我没写对？Yoda 把这条链路整个摊开，逐项查看、按需覆写。多数"agent 怎么突然不行了"，根子不在模型，就在这层没人给你看的 harness 里。

有人说我偏执，做个工具张口闭口都是 harness。是的，我这些年就认这一件事——harness。我没有别的指望，就一个指望：让它不再是被忽略的术语、"模型决定一切"的空话，而成为你每天实实在在的工作方式，成为你和你那队 agent 之间牢不可破的默契。

这，就是 Yoda：**Your Orchestra of Delegated Agents**——你那支被妥善指挥的 agent 乐队。我深信，总有一天，"指挥一队 agent 干活"会像日常聊天一样，简单自然。

不绑模型，不绑厂商，本地优先。路虽远，行则尤达。这一天，不必再等——愿原力与你、与你的 agent 乐队同在。
